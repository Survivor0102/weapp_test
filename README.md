# 待办清单小程序 📝

一个简洁高效的微信小程序待办事项管理应用。

## 📁 项目结构

```
todo-miniprogram/
├── app.js                    # 全局入口
├── app.json                  # 全局配置
├── app.wxss                  # 全局样式
├── project.config.json       # 项目配置
├── sitemap.json             # 搜索索引配置
│
├── pages/                   # 页面目录
│   ├── index/               # 首页 - 待办列表
│   ├── add/                 # 新增/编辑页
│   └── category/            # 分类管理页
│
├── components/              # 公共组件
│   ├── todo-item/           # 待办项组件
│   ├── filter-bar/          # 筛选栏组件
│   └── empty-state/         # 空状态组件
│
├── utils/                   # 工具函数
│   ├── storage.js           # 本地存储封装
│   ├── util.js              # 通用工具
│   └── date.js              # 日期处理
│
└── styles/                  # 样式变量
    └── variables.wxss       # CSS 变量定义
```

## 🚀 快速开始

### 1. 导入项目
1. 打开微信开发者工具
2. 选择「导入项目」
3. 选择项目目录：`todo-miniprogram`
4. 填写 AppID（测试可选「测试号」）
5. 点击「导入」

### 2. 配置项目
- 修改 `project.config.json` 中的 `appid` 为你的小程序 AppID
- 根据需要调整 `app.json` 中的页面配置和窗口样式

### 3. 运行项目
- 点击微信开发者工具的「编译」按钮
- 在模拟器或真机上预览

## 📖 使用说明

### 添加待办
1. 点击首页右下角「+」按钮
2. 输入待办内容（必填）
3. 选择分类（可选）
4. 设置优先级（可选）
5. 设置截止时间（可选）
6. 点击「保存」

### 完成待办
- 点击待办项左侧的复选框

### 编辑待办
- 点击待办项内容区域

### 删除待办
- 点击待办项右侧的「删除」按钮
- 确认删除

### 筛选待办
- 点击顶部筛选栏：全部 / 未完成 / 已完成 / 分类
- 选择具体分类进行筛选

### 管理分类
1. 点击筛选栏的「分类」
2. 进入分类管理页面
3. 点击「+ 添加分类」
4. 输入分类名称，选择颜色
5. 点击「保存」

## 🎨 样式定制

所有样式变量定义在 `styles/variables.wxss` 中，可快速定制主题：

```css
page {
  --primary-color: #3498db;      /* 主题色 */
  --success-color: #2ecc71;      /* 成功色 */
  --danger-color: #e74c3c;       /* 危险色 */
  --bg-color: #f5f6fa;           /* 背景色 */
  --card-bg: #ffffff;            /* 卡片背景 */
  /* ... 更多变量 */
}
```

## 💾 数据存储

使用微信小程序本地存储：
- `todos` - 待办事项数组
- `categories` - 分类数组

数据自动持久化，无需担心丢失。

## 📦 数据格式

### 待办事项
```javascript
{
  id: 'id_1711353600000_abc123',
  title: '完成项目报告',
  completed: false,
  categoryId: 'cat_001',
  priority: 'high',        // high | medium | low
  dueDate: 1711353600000,  // 时间戳
  createdAt: 1711267200000,
  updatedAt: 1711267200000
}
```

### 分类
```javascript
{
  id: 'cat_001',
  name: '工作',
  color: '#3498db',
  icon: 'work'
}
```

## 📝 开发规范

### 代码风格
- 使用 ES6+ 语法
- 组件化开发
- 样式使用 CSS 变量
- 注释清晰完整

### 命名规范
- 文件名：小写 + 中划线（如 `todo-item.js`）
- 变量名：小驼峰（如 `todoList`）
- 常量名：大写下划线（如 `TODO_KEY`）
- 组件名：小驼峰（如 `todoItem`）
